마인드셋 전환 1

"코드 생성기"가 아니라 "사고 파트너"

흔한 오해

  • AI는 코드 자동완성 도구다
  • 완벽한 프롬프트를 찾아야 한다
  • 한 번에 전체를 요청해야 한다
  • AI가 틀리면 내가 직접 고친다

효과적인 접근

  • AI는 대화하며 함께 문제를 푸는 파트너
  • 대화를 통해 점진적으로 발전시킨다
  • 작은 단위로 나누어 반복한다
  • 에러 발견 시 AI에게 재작성을 지시한다

전동 공구가 등장해도 목수의 전문성이 사라지지 않듯,
AI 도구의 등장이 개발자의 가치를 대체하지 않습니다.

PRACTICE 1

맥락의 차이 체험하기

Step 2: 맥락 제공
"삼성 SSD FW에서 NVMe SMART 로그 파싱 필요

- NVMe Spec 2.0, Log Page ID 02h 기준
- C99, 동적 할당 금지, 빅엔디안 처리
- Critical Warning, Temperature 추출
- 에러 시 음수 반환"

추가 실험: Step 2 결과에 "이 코드 리뷰해줘. race condition, 메모리 정렬 확인" 요청
